#+title: Changelog

All notable changes to org-dex-parse are documented in this file.

Format follows [[https://keepachangelog.com/en/1.1.0/][Keep a Changelog]].
Versioning follows [[https://semver.org/spec/v2.0.0.html][Semantic Versioning]].

* [Unreleased]

** Added
- L12 post-parse predicate check (S19g): headings matching the item predicate
  but missing =:ID:= now produce a =ParseWarning= in =ParseResult.warnings=.
  Only fires when an explicit predicate is configured (default predicate skips
  the check to avoid noise).  New types: =ParseWarning= dataclass,
  =Config.predicate_is_default= flag.

** Fixed
- Input validation: non-string operators in predicates (e.g. a nested list
  where a string was expected) now raise =ValueError= instead of =TypeError=
- Input validation: non-string property names in predicates (e.g.
  =["property", 1]=) now raise =ValueError= at compile time instead of being
  silently accepted
- Input validation: non-string elements in config collection fields (e.g.
  =exclude_drawers: [1]=) now raise =ValueError= instead of =AttributeError=
- CLI: error message for invalid configuration generalized from "invalid
  predicate" to "invalid configuration" to cover non-predicate validation
  errors

* [0.1.2] — 2026-03-17

** Fixed
- Timestamp property robustness: impossible dates (e.g. Feb 31) in =CREATED=
  or =ARCHIVE_TIME= properties now return =None= instead of crashing with
  =ValueError=; mismatched delimiters (e.g. =<2026-03-01]​=) are also
  rejected as malformed
- Tag regex restore: =parse_file= now restores orgparse's global
  =RE_HEADING_TAGS= regex after every call (via =try/finally=), preventing
  a failed parse from contaminating subsequent calls in the same process
- Clock ordering: =Item.clock= now sorts by start time, guaranteeing
  chronological order even when clock entries are interleaved across
  the item node and its scaffolding children
- Bare URL balanced parentheses: URLs like
  =https://en.wikipedia.org/wiki/Foo_(bar)= are now preserved intact
  (previously the closing paren was stripped, breaking the link)
- CLI: malformed JSON in config file now produces a user-friendly error
  instead of a Python traceback
- CLI: malformed =--predicate= JSON and invalid predicate expressions now
  produce a clear error instead of a Python traceback

* [0.1.1] — 2026-03-16

** Fixed
- Bare URL trailing dot: =https://example.com/page.= in prose now strips
  the final period (previously included it in the link target)
- CLI: =--todos "TODO, NEXT,"= now strips whitespace and drops empty tokens
  (previously produced =" NEXT"= and =""=)
- CLI: JSON config with scalar where list expected (e.g. ="todos": "LOGBOOK"=)
  now fails with a clear error instead of silently iterating the string

** Changed
- CLI rewritten as generic public tool (=python -m org_dex_parse=)
- Adapter layer isolates orgparse private API access
- Pre-computed item map for O(1) is_item lookup during tree walks
- Link deduplication uses bisect instead of linear scan
- Removed unused =parent_map= parameter from =_build_inherited_tags_map=

* [0.1.0] — 2026-03-16

Initial release.

** Added
- Parse org-mode files into structured =Item= dataclasses (24 fields)
- Configurable item predicate via s-expression evaluator
- Zone-aware extraction: body, links, timestamps, clock entries, state changes,
  properties, tags (with inheritance), raw text
- Parent cache for 4x speedup on large files (eliminates O(n^2) bottleneck)
- CLI for exploration (=python -m org_dex_parse=)
- Comprehensive test suite (14 modules, 100+ tests)
